## Step 1️⃣: Google Cloud Console にアクセス

### 1-1: ブラウザで URL を開く

```
URL を入力:
https://console.cloud.google.com/

または Google で検索:
「Google Cloud Console」→ 公式サイトをクリック
```

### 1-2: Google アカウントでログイン

```
ページに到着後、右上に「ログイン」ボタンが表示されます。

クリック → Google ログイン画面が出現

入力:
┌─────────────────────────────────┐
│ メールアドレス: [CEO 神崎慧のメール] │
│                                 │
│ パスワード: [パスワード]          │
│                                 │
│ 「次へ」をクリック                │
└─────────────────────────────────┘

※2FA（二段階認証）が設定されている場合:
  - スマートフォンに確認コードが送信されます
  - コードを入力してください
```

### 1-3: ログイン完了

```
Google Cloud Console のダッシュボードが表示されます。
左側に「Google Cloud」というロゴが見えたら成功です。
```

## 🔒 セキュリティに関する注意

### シークレットキーの扱い

```
⚠️ 絶対にしてはいけないこと：

❌ シークレットキーをメール送信
❌ シークレットキーをチャットに投稿
❌ シークレットキーをソースコードに記述（公開リポジトリ）
❌ シークレットキーを Screenshot で撮影して共有
❌ シークレットキーを SMS で送信

✅ 安全な保管方法：

✓ パスワードマネージャー（1Password, Bitwarden 等）
✓ Discord の秘密チャネル
✓ 暗号化されたファイル
✓ CEO のローカルマシンのみ
```

### サイトキーについて

```
✅ サイトキーは「公開」で OK

理由:
• フロントエンド（HTML）に記述されるため
• 隠す必要がない
• 盗まれても被害がない（スコアが不正にならない程度）

用途:
• Contact Form の HTML で使用
• reCAPTCHA JavaScript に渡す
```

## 🆘 トラブルシューティング

### Q: キーが作成されない

```
A: 以下を確認してください：

1. reCAPTCHA Enterprise API が有効になっているか？
   → API とサービス → 有効な API 確認

2. プロジェクトが正しく選択されているか？
   → 画面左上でプロジェクト名確認

3. ドメインが正しく登録されているか？
   → キー作成時にドメイン 4 個確認

4. reCAPTCHA v3 が選択されているか？
   → v2 ではなく v3 を選択
```

### Q: シークレットキーを忘れた

```
A: Google Cloud Console で再確認：

1. API とサービス → reCAPTCHA
2. 作成済みキー一覧から該当キーを選択
3. シークレットキーを「表示」して確認
4. 再度コピーして保管
```

### Q: キーを失効させたい

```
A: キーの削除・再作成：

1. API とサービス → reCAPTCHA
2. キー一覧から削除対象を選択
3. 「削除」をクリック
4. 新規キーを作成

※ 削除後、サイトは再度認証が必要になります
```
