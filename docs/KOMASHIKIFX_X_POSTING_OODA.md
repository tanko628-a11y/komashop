**最終更新**: 2026-04-23

---

## 📋 プロジェクト概要

### 目的

MQL4 指標開発（kr_v3、karrows、MyChannel_VWAP など）の「開発プロセス」を透明に公開し、以下を実現する：

```
開発プロセス公開
    ↓
共感 → 信用獲得
    ↓
テスター獲得 → 販売促進
```

### 対象

- **ターゲット**: FX トレーダー・開発者（潜在テスター・購買層）
- **効果**: 販売促進・テスター獲得
- **投稿頻度**: 月間 20～25 投稿

## 🔄 OODA サイクル：開発日記を通じた改善フロー

### 実行時間帯

**毎晩 23:00～02:00** に以下の OODA サイクルを実行

### ① Observe（観察）

```
「同ジャンル（FX/先物）で何がバズっているか」

分析対象:
  - テスター成績型が最高反応？
  - 失敗学型が信用獲得？
  - 相場実況型が毎日の基盤？

データ:
  - 過去 7 日分の投稿反応を分析
  - タイプ別のいいね数・リーチを記録
```

### ② Orient（方向づけ）

```
「FX ジャンルの『新しい風』は何か」

差別化要因:
  - チャート画像なし（法的リスク回避 + 秘匿）
  - 統計性重視（勝率ではなく絶対値）
  - 失敗も含める（透明性MAX）
  
結論:
  「インチキなし」が差別化要因
```

### ③ Decide（決定）

```
「明日の投稿タイプを決める」

判断基準:
  - 高反応だったタイプを翌日も試す
  - または新しいタイプにチャレンジ

例:
  テスター成績型が伸びてたら
  → 翌日も同型を投稿
```

### ④ Act（実行）

```
「投稿テンプレを準備」

手順:
  1. Gmail 下書きから該当テンプレを選択
  2. 数字を最新版に更新
  3. X_投稿ガイドラインで最終チェック
  4. 翌日 17:00 に投稿実行
```

### ⑤ Reflect（反省）

```
「朝レポで成功要因をメモ」

記録内容:
  - 「テスター成績型が +247 いいね獲得」
  - 「理由は統計性を前に出した点」
  - 「来週も同タイプを集中投下」

出力:
  → Discord 朝レポに自動投稿
```

## ⚖️ 法的安全性：金融商品取引法対応

### 【絶対NG】❌

```
❌ チャートを見せてエントリーポイント示唆
❌ 「買い目線」「売り目線」を明示
❌ 「勝率 XX%」の強調
❌ 「このツール買うと勝てます」
❌ 「確実に儲かる」
```

### 【セーフ】✓

```
✓ 「自分は +150pips 達成」（過去実績報告）
✓ 「テスター成績：+58,500 円/人」（統計根拠）
✓ 「失敗も経験した」（透明性）
✓ 「詳しくはショップで」（情報提供）
```

### 【パクリ防止】🔒

```
✓ チャート画像なし
✓ ロジック詳細なし
✓ パラメータ値なし
✓ テストケースの具体数値なし

→ 「開発姿勢」だけ見せて、ロジックは秘匿
```
