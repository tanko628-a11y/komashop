# 🎯 X投稿実装ガイド — 実行マニュアル

**目的**: OODA ループを実装し、日常的な X 投稿を効率化  
**対象**: 凪フィナンシャル X投稿オートメーション  
**実装開始**: 2026-04-24

---

## 📊 OODA サイクル概要

```
【毎日のサイクル】

23:00 ← OBSERVE（自動実行）
  ↓ 相場観察 & トレンド分析
  ↓ Discord に観察結果を Post
  ↓ ユーザーにメール候補を提示

[ユーザー: 5分の判断]
  ↓ メールテンプレートを選択

06:00 ← REFLECT（自動実行）
  ↓ 前日の投稿をすべて分析
  ↓ エンゲージメント集計
  ↓ Discord に朝レポート Post
  ↓ 改善提案を表示

17:00 ← ACT（ユーザー手動）
  ↓ 選択したメール内容を確認
  ↓ X（Twitter）に投稿
  ↓ 4時間ごとに再投稿（オプション）

[毎日自動で回転]
```

## ✅ 実装ステップ（優先度順）

### フェーズ 1: セットアップ（Week 1）

#### **Step 1.1: Google Apps Script プロジェクト作成** ⭐ START HERE

```
1. Google Drive → 新規作成 → Google Apps Script
   プロジェクト名: "凪FX - X投稿 OODA 自動化"

2. エディタを開く

3. 以下の 2 つの関数をコピー・ペースト
   - observeMarket() [GAS_X_POSTING_SETUP.md から]
   - reflectOnPostings() [GAS_X_POSTING_SETUP.md から]

4. 保存（Ctrl+S / Cmd+S）
```

**予定時間**: 15分

---

#### **Step 1.2: API キー・認証情報設定**

```
1. Google Apps Script → プロジェクト設定
   → スクリプトプロパティ → 編集

2. 以下の 4 項目を追加:

DISCORD_WEBHOOK_URL
  値: https://discord.com/api/webhooks/[YOUR_WEBHOOK_ID]/[YOUR_WEBHOOK_TOKEN]

TWITTER_BEARER_TOKEN
  値: AAAA... (Twitter Developer Portal から取得)

MT4_API_URL
  値: https://api.example.com/mt4 (または実装中なら 'mock')

GMAIL_LABEL
  値: "X投稿/相場実況型"
```

**セキュリティ注意**: 
- API キーをコード内に書かない（スクリプトプロパティを使用）
- GitHub にコミットしない（.gitignore に追加）

**予定時間**: 10分

## 🔧 トラブルシューティング

### 問題 1: Observe が 23:00 に実行されない

```
【原因】トリガーが作成されていない或いは無効

【確認】
1. Google Apps Script → トリガー
2. observeMarket トリガーが存在するか確認
3. 有効か確認（チェックボックス）

【解決】
1. 無ければ新規作成（Step 1.5 参照）
2. タイムゾーン: Asia/Tokyo を確認
3. 実行ログを確認: 実行履歴
```

---

### 問題 2: Discord に投稿されない

```
【原因】Webhook URL が正しくない或いは権限不足

【確認】
1. スクリプトプロパティの DISCORD_WEBHOOK_URL を確認
2. 末尾に余分なスペース/改行がないか
3. Discord サーバーで Webhook が削除されていないか

【解決】
1. Discord で新規 Webhook を作成
2. URL をコピー（余分な文字なし）
3. スクリプトプロパティを更新
4. テスト実行
```

---

### 問題 3: API キーエラー（Twitter API）

```
【確認】
1. Twitter Developer Portal にアクセス
2. Bearer Token が有効か確認
3. Regenerate していないか

【解決】
1. 新規トークンを生成
2. スクリプトプロパティに登録
3. テスト実行
```
