## 📊 PDCA サイクルの定義

```
┌─────────────────────────────────────────────────────┐
│              🔄 PDCA サイクル（4週間）              │
├─────────────────────────────────────────────────────┤
│                                                     │
│ 📋 PLAN（月曜朝）                                   │
│  └─ 今週の目標設定 + 実行計画書作成                 │
│                                                     │
│ ✅ DO（火〜木曜）                                    │
│  └─ 計画通り実行 + 日々の進捗記録                   │
│                                                     │
│ 📊 CHECK（金曜夜）                                  │
│  └─ 実績測定 + 目標達成度判定                       │
│  └─ 失敗・ブロッカー分析                            │
│                                                     │
│ 🔧 ACT（土曜）                                      │
│  └─ 改善策立案 → 翌週の計画に反映                   │
│  └─ 同じ失敗は二度としない体制構築                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**成果物:** 週間計画書

**やること:**

```

   └─ 前週の失敗を回避するための工夫を明記

2️⃣ 今週のマイルストーン設定
   ├─ 柱1（NK225シグナル）: 目標値
   ├─ 柱2（MT4インジケーター）: 目標値
   ├─ 柱3（SEOメディア）: 目標値
      ```

      ```

4️⃣ リスク・ブロッカー予測
   ├─ 「先週できなかったことは何か」を分析
   ├─ 同じことが今週起きないための対策を記載
   └─ 例: 
      ```
      【先週の失敗】XServer サポート対応が遅い

### 金曜 18:00 - CHECK フェーズ

**責任者:** CEO かんざき けい + オーナー小松直人（監督）  
**所要時間:** 2時間  
**成果物:** 週次実績レポート + 失敗分析書

**やること:**

```
1️⃣ KPI 実績測定（1時間）
   ├─ 月曜に設定した目標値 vs 実際の成果値
   ├─ 達成度を % で算出
   └─ 表形式で Discord に投稿
   
例:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
| 指標 | 目標 | 実績 | 達成度 | 評価 |
|-----|------|------|--------|------|
| NK225 売上 | ¥100K | ¥60K | 60% | ❌ |
| MT4 販売 | 5本 | 3本 | 60% | ❌ |
| SEO PV | +100 | +50 | 50% | ❌ |
| 全体 | - | - | 57% | ⚠️ |
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2️⃣ 失敗・ブロッカー分析（30分）
   ├─ 「目標達成できなかった理由」を根本分析
   ├─ 5段階なぜ分析を実施
   └─ 例:
      ```
      【失敗】NK225 売上が目標¥100Kに対し実績¥60K
      
      ① なぜ売上が低い？
         → ペーパートレードの実績公開が遅れた
      
      ② なぜ実績公開が遅れた？
         → トレード記録の手作業が時間かかった
      
      ③ なぜ手作業が時間かかった？
         → Google Sheets の記入が面倒だった
      
      ④ なぜ面倒だった？
         → 自動化できていなかった
      
      ⑤ どうする？
         → Google Apps Script で自動記録化
      ```

3️⃣ 改善策立案（30分）
   ├─ 原因に対して「再発防止策」を設定
   ├─ 翌週の計画に反映させる形で記載
   └─ 例:
      ```
      【改善策】
      - Google Apps Script で自動トレード記録化（来週実装）
      - komashikifx.site に「本日のトレード」を自動投稿化
      → 結果: 運用時間 80% 削減 + PV増加
      ```

4️⃣ 成功事例の共有
   ├─ 「今週成功したこと」を記録
   ├─ その理由を分析
   └─ 来週以降も継続する
      ```
      【成功事例】
      - SEO 記事2本投稿 → +50PV
      - 理由: キーワード選定が良かった
      - 来週: 同じキーワード戦略で3本投稿予定
      ```
```

### 土曜 10:00 - ACT フェーズ

**責任者:** オーナー小松直人（最終決定）  
**所要時間:** 1時間  
**成果物:** 改善実行計画 → 月曜の PLAN に反映

**やること:**

```
1️⃣ CEO レポートを精読（30分）
   ├─ 金曜の CHECK レポートを読む
   ├─ 失敗分析・改善策を理解する
   └─ 質問・意見があれば返信

2️⃣ 改善策の承認・調整（20分）
   ├─ 「その改善策で本当に改善するか」を判定
   ├─ 予算が必要な場合は承認
   └─ 翌週の計画に組み込み指示

3️⃣ 翌週への改善指示を Slack/Discord へ投稿
   ├─ オーナーからの承認メッセージ
   ├─ 月曜 PLAN に反映すべき項目を明記
   └─ 例:
      ```
      【ACT】オーナー承認・指示
      
      ✅ CEO の改善策「Google Apps Script自動化」を承認
      実装期限: 来週水曜まで
      予算: なし（時間だけ）
      
      ⚠️ 「人員採用」の検討: 月売上¥100K達成後 → 承認
      予算上限: 月¥300,000（給与）
      
      💡 アイディア: MT4バックテストの自動化も検討しては？
      
      次週の PLAN への反映項目:
      - Google Apps Script 実装（水曜まで）
      - バックテスト自動化の検討（金曜まで）
      ```

4️⃣ 月曜 PLAN を CEO に指示
   ├─ 改善策をどう計画に組み込むか指示
   └─ 月曜朝 09:00 に新しい PLAN を提出するよう指示
```

**Discord への投稿:**
